# Net Positive Skillify Update

Makes surgical changes to an existing skill, only after the user has approved
them. Changes arrive by two routes: the confirmed tier of a net-positive-skillify:review
report, or a direct user request (a rename, a fix, a new reference, a workflow
tweak) with no review involved. Both routes use the same machinery and the same
discipline. Two rules govern every change:

1. **Code over the model.** Prefer adding or extending a script so a
   deterministic job (scoring, formatting, assembling data, patching an
   artifact) is done by code, not regenerated by the model each run.
2. **Surgical, not wholesale.** Make the smallest edit that achieves the change.
   Never reproduce a whole file to alter part of it. Edits are exact
   find/replace operations, file additions, or renames, applied by a script that
   fails loudly if a target is not matched.

When applying a review, only the **confirmed** tier is ever applied. Uncertain
findings go back to a human; suppressed findings are not findings. Direct
requests need no review first; the user's approval of the specific edits is the
gate.

## Workflow

```
- [ ] Step 1: Establish the changes (review route or direct request) and confirm approval
- [ ] Step 2: Seed the patch (verdicts_to_patch.py from a review; by hand otherwise)
- [ ] Step 3: Author the edits (fill in find/replace, add files)
- [ ] Step 4: Dry-run, then apply (run apply_patch.py)
- [ ] Step 5: Verify (re-run the review, or targeted checks for direct edits)
- [ ] Step 6: Repackage the skill if needed and report what changed
```

**Step 2: Seed the patch.** On the review route, if the review produced a
`verdicts.json`, convert
its confirmed tier into a patch skeleton deterministically:

```bash
python3 scripts/verdicts_to_patch.py verdicts.json <target-skill-dir> patch.json
```

Each confirmed recommendation becomes a stub edit carrying its file, line, kind,
and recommendation text, with the `find` and `replace` left for authoring. On
the direct route there is nothing to seed: write `patch.json` from scratch, one
edit per requested change, and restate the planned edits to the user before
applying if the request left any room for interpretation.

**Step 3: Author the edits.** Translate each approved change into the
smallest concrete edit. Patch schema and worked examples:
[references/patching.md](references/patching.md). For a recommendation to "use
code instead of the model", the edit usually adds a script (`add_file`) and
replaces the prose step in a workflow file with an instruction to run it. Read
the target file immediately before quoting it; a `find` string must match the
file exactly.

**Step 4: Dry-run, then apply.**

```bash
python3 scripts/apply_patch.py patch.json --dry-run
python3 scripts/apply_patch.py patch.json
```

The dry run validates every edit (files exist, find strings match the expected
count) without writing anything, so a whole patch is proven landable before a
single byte changes. Each find/replace must match an exact, unique string, so an
edit either lands precisely or fails without changing anything. Fix any reported
failure and re-run; do not fall back to rewriting the file by hand.

**Step 5: Verify.** On the review route, re-run the net-positive-skillify:review pipeline
against the updated skill and confirm the targeted findings are cleared:
structural flags gone, and previously confirmed efficiency findings now absent
(or correctly suppressed by the verify step because the behaviour has changed).
On the direct route, view the changed lines to confirm each edit landed as
intended, and run the review's structure analyser as a cheap regression check.

## Inputs and outputs

- **Input:** the approved changes, either the confirmed tier of a review's
  `verdicts.json` or a direct user request, plus the target skill.
- **Intermediate:** `patch.json` (the targeted edits).
- **Output:** the updated skill, changed in place by the smallest edits possible.

## Dependencies

- `python3` (standard library).
- `net-positive-skillify:review` for Step 5 verification (recommended, not required).
